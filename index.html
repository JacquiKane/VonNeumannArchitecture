Von Neumann Architecture

In 1946, John Von Neumann (principal author), Arthur Burks and Herman Goldstine described a theoretical design for a computer that stored programs. Prior to this, large computers were programmed manually, using switches - so the ability to store instructions was a huge innovation.

Data and instructions stored together, if the instructions could be stored in the same electronic memory that held the data, they could be accessed as quickly as needed. This made High Level Languages (HLL) possible.

  • an ALU (Arithmetic and Logic Unit)
  • a controller
  • the instruction fetch cycle

Von Neumann architecture became the basis for modern computer architecture.

Modern computers include memory, a processor (encompassing the logic unit and controller), a distinct fetch cycle, and mechanisms for input and output.

The ENIAC(Electronic Numerical Integrator & Computer) was the first electronic programmable computer.The ENIAC had to be rewired to solve different problems.

The EDVAC(1949), Electronic Discrete Variable Automatic Computer, was a successor to the ENIAC. John Von Neumann's document, First Draft of a Report on the EDVAC, proposed the "stored-program" concept that embodies the Von Neumann architecture - storing of the program in the same memory as the data. The British computers EDSAC at Cambridge and the Manchester Baby were the first working computers that followed this design. And it has been followed by the great majority of computers made since. Having the program and data in different memories is called the Harvard architecture.

Storage

Memory

Memory or storage in the Von Neumann architecture, was spread throughout the device. It provided a mechanism to store both instructions and data.

The data needed by key instructions had become readily available - it was stored in the same memory as the instructions. This was a milestone that made processing time much faster.

Arithmetic & Logic Unit

ALU

The logic unit in the Von Neumann architecture, was comprised of registers and logic units for processing the operations.

The registers stored and accessed both the instructions and the data retrieved from memory, and the logic circuitry processed the operation as defined by the sequence of instructions.

Controller

Controller

The controller in the Von Neumann architecture, comprised of an instruction register and a program counter to sequence the operations.

This critical component worked with the ALU to ensure that all of the instructions were processed in the order required for successful completion.

Interpreting Instructions

  • The controller and the ALU are subsumed into the processor.
  • Data and instructions are stored in memory.
  • Instructions are fetched from memory one at a time and in order (serially).
  • The processor decodes and executes an instruction, before cycling around to fetch the next instruction.

Input, Output

IO

Von Neumann architecture supported the key functions of input (getting data and instructions into the computer) and output (communicating the result of a computation).

Solving a problem and providing the results are the core requirements of any computing device.Input provides a way to alter the data being processed, so that the process works for multiple data sets.

First Draft - Stored Program

1945, First Draft Of A Report On The EDVAC

The First Draft of a Report on the EDVAC (commonly shortened to First Draft) is an incomplete 101-page document written by John von Neumann and distributed on June 30, 1945 by Herman Goldstine, who worked on the classified ENIAC project. It contains the first published description of the logical design of a computer using the stored-program concept, which became known as the Von Neumann architecture.
